이제 User Prompt를 설계하는 3가지 방법을 직접 실험해 봅시다.

- **Zero-shot**: 예시 없이 바로 요청
- **Few-shot**: 예시를 먼저 보여주고 패턴 유도
- **CoT**: 단계적 사고를 유도해 추론 안정성 높이기

아래 실습은 모드별 예시 프롬프트를 보고, 사용자가 직접 프롬프트를 작성해 실행하는 방식입니다.

## 실습 포인트

1. Zero-shot과 Few-shot의 출력 일관성 차이를 비교하기
2. CoT가 계산/추론 문제에서 어떤 도움을 주는지 관찰하기
3. 같은 모델이라도 프롬프트 설계가 성능 체감에 큰 영향을 준다는 점 이해하기
